# Proj0052: MSBuild property could not be resolved by the analyzer
Properties referenced via `$(...)` in path expressions are resolved by the
analyzer when they are reserved (e.g. `MSBuildThisFileDirectory`),
registered via `<CompilerVisibleProperty>`, or unconditionally defined in
`<PropertyGroup>`. Otherwise the reference is silently skipped (no
false-positive diagnostic on the path-existence rule). Enabling this
informational rule surfaces those skipped references so they can be
registered or refactored.

This rule is `isEnabled: false` by default. Opt in via `.editorconfig`:

``` ini
[*.csproj]
dotnet_diagnostic.Proj0052.severity = suggestion
```

## Triggered
The property `$(SomeUnknownProperty)` is not reserved, not registered via
`<CompilerVisibleProperty>`, and not defined in any unconditional
`<PropertyGroup>`.
``` xml
<Project Sdk="Microsoft.NET.Sdk">

  <ItemGroup>
    <ProjectReference Include="$(SomeUnknownProperty)/Foo.csproj" />
  </ItemGroup>

</Project>
```

## Compliant
The property is defined unconditionally and the analyzer resolves it.
``` xml
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <CustomDir>some/path</CustomDir>
  </PropertyGroup>

  <ItemGroup>
    <ProjectReference Include="$(CustomDir)/Foo.csproj" />
  </ItemGroup>

</Project>
```
