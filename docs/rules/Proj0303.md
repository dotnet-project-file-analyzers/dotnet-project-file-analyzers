# Proj0303: Define mappings for multiple sources
Explicit declaration of package source mappings ensures reproducible builds and
mitigates the risk of [supply chain attacks](https://cheatsheetseries.owasp.org/cheatsheets/Software_Supply_Chain_Security_Cheat_Sheet.html).
The reasoning is that, when mulitple sources are configured, by having mappings
defined it is unambiguous which source provides which package.

## Non-compliant
``` xml
<configuration>

  <packageSources>
    <clear />
    <add key="Internal-packages" value="https://pkgs.dev.azure.com/company/_packaging/Components/nuget/v3/index.json" />
    <add key="nuget.org" value="https://api.nuget.org/v3/index.json" />
  </packageSources>

</configuration>
```

## Compliant
``` xml
<configuration>

  <packageSources>
    <clear />
    <add key="Internal-packages" value="https://pkgs.dev.azure.com/company/_packaging/Components/nuget/v3/index.json" />
    <add key="nuget.org" value="https://api.nuget.org/v3/index.json" />
  </packageSources>

  <packageSourceMapping>
    <packageSource key="nuget.org">
      <package pattern="*" />
    </packageSource>
    <packageSource key="Internal-packages">
      <package pattern="Qowaiv.CodeGeneration" />
      <package pattern="Company.*" />
    </packageSource>
  </packageSourceMapping>

</configuration>
```
