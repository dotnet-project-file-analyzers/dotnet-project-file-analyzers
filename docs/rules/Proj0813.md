# Proj0813: Global package references only work when CPM is enabled
When using [Central Package Management](Proj0800.md),
it is possible to define (private) global package references with the use of
[`<GlobalPackageReference>`](https://learn.microsoft.com/nuget/consume-packages/central-package-management#global-package-references) in the `Directory.Packages.props` file.

If CPM is not enabled, however, `<GlobalPackageReference>`'s are ignored.

## Non-compliant
``` xml
<Project>

  <PropertyGroup>
    <ManagePackageVersionsCentrally>false</ManagePackageVersionsCentrally>
  </PropertyGroup>

  <ItemGroup>
    <GlobalPackageReference Include="DotNetProjectFile.Analyzers" Version="1.11.0" />
  </ItemGroup>

</Project>
```

## Compliant
``` xml
<Project>

  <PropertyGroup>
    <ManagePackageVersionsCentrally>true</ManagePackageVersionsCentrally>
  </PropertyGroup>

  <ItemGroup>
    <GlobalPackageReference Include="DotNetProjectFile.Analyzers" Version="1.11.0" />
  </ItemGroup>

</Project>
```
