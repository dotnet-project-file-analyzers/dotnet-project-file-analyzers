# Proj1303: Avoid project references to bencmark projects
The use of multiple projects in a single solution is to improve code organization by
separating concerns into distinct layers or components, making maintenance and
testing easier, facilitating team collaboration, and enabling the reuse of code
across different applications.

[Benchmark projects](https://benchmarkdotnet.org/) should not have another purpose
than just be executed to run the containing benchmarks, and therefor should not
be referenced by other projects.
