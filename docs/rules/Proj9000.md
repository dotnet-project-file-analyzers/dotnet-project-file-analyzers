# Proj9000: MSBuild property could not be resolved by the analyzer
Properties referenced via `$(...)` in path expressions are resolved by the
analyzer when they are reserved (e.g. `MSBuildThisFileDirectory`),
registered via `<CompilerVisibleProperty>`, or unconditionally defined in
`<PropertyGroup>`. References that resolve to none of those are silently
skipped, so the path-existence rules do not produce a false positive.

This rule surfaces those skipped references for analyzer-internal
diagnostic use. It is hidden and disabled by default; enable it via
`.editorconfig` when investigating which property references the analyzer
is unable to resolve:

``` ini
[*.csproj]
dotnet_diagnostic.Proj9000.severity = suggestion
```

## Triggered
The property `$(SomeUnknownProperty)` is not reserved, not registered via
`<CompilerVisibleProperty>`, and not defined in any unconditional
`<PropertyGroup>`.
``` xml
<Project Sdk="Microsoft.NET.Sdk">

  <ItemGroup>
    <ProjectReference Include="$(SomeUnknownProperty)/Foo.csproj" />
  </ItemGroup>

</Project>
```

## Compliant
The property is defined unconditionally and the analyzer resolves it.
``` xml
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <CustomDir>some/path</CustomDir>
  </PropertyGroup>

  <ItemGroup>
    <ProjectReference Include="$(CustomDir)/Foo.csproj" />
  </ItemGroup>

</Project>
```
